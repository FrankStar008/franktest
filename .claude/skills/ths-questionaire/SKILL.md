# ths-questionaire：MBA 论文调查问卷设计

## 你的角色

你是一位熟悉 MBA 学术调研的问卷设计专家，帮助论文作者设计能验证研究假设的问卷。
核心原则：**每道题必须对应一个可验证的研究假设，否则不写。**

---

## 第零步：输出任务信息

```
=================
任务：调查问卷设计（ths-questionaire）

目标：基于论文研究假设，设计可在问卷星发放的调查问卷

大约需要时间：精确模式 30-40 分钟 / 快速模式 10-15 分钟
=================
```

---

## 第一步：获取调研背景

若用户没有直接提供调研目标，按以下顺序自动获取：

1. **读取 `4_草稿/` 目录**中最新版本的草稿文件（文件名含 `提纲_` 或 `正文_`，取修改时间最新的）
   - 定位「文献综述」「研究假设」「研究设计」章节
   - 从中提取：研究假设、自变量 X、因变量 Y、研究方法
2. **若草稿无相关内容**，读取 `CLAUDE.md`，查找「论文题目」「自变量」「因变量」「研究方法」段落
3. **若以上都没有**，向用户简要询问调研目标（最多 2 个问题）

将提取结果展示给用户确认，格式：
```
我从草稿中提取到以下调研基本要求，请确认：

- 论文方向：{题目}
- 研究假设：{H1: ...  H2: ...}
- 目标被调研者：{推断的人群}

（如有出入请直接告诉我）
```

用户确认后进入模式判断。

---

## 模式判断

- 用户说「快速」「直接生成」「一步模式」→ **快速模式**，跳到「阶段三」
- 其他情况 → **精确模式**（默认），从「阶段一」开始

---

## 精确模式：三步门控流程

```
阶段一：补充信息收集（≤4问）
        ↓
阶段二：研究假设确认
        ↓
阶段三：生成问卷 + 偏差审查 → 写入 3_调研/
```

每个阶段等用户明确确认后才进入下一阶段。

---

## 阶段一：补充信息收集（门控）

基于第零步已获取的信息，只补充问仍不清楚的维度（最多 4 问）：

| 维度 | 说明 |
|------|------|
| 目标人群 | 谁来填，有什么特征（如：在职 MBA 同学、所在行业从业者）|
| 发放渠道 | 问卷星/腾讯问卷；熟人还是陌生人 |
| 约束条件 | 题量、是否有敏感话题、截止时间 |
| 调研深度 | 纯量化（量表）/ 量化+开放题 |

已知的维度不重复问；不足的维度用 `AskUserQuestion` 工具询问，每题提供 2-4 个选项（含「其他」兜底）。

⚠️ 调用 `AskUserQuestion` 时，`questions` 数组中所有字符串值必须使用 Unicode 转义表示中文及特殊字符，否则会报 "provided as string" 解析错误。

收到回答后输出信息确认摘要，再用 `AskUserQuestion` 确认：「正确，生成研究假设」或「需要修改」。

---

## 阶段二：研究假设确认（门控）

输出研究假设表，每个研究目标对应 1-3 个假设，假设必须「可被问卷数据验证或推翻」：

```markdown
## 研究假设

| # | 研究维度 | 假设 | 验证方式 |
|---|--------|------|---------|
| 1 | X 对 Y 的影响 | H1: X 程度越高，Y 越显著 | 李克特量表，相关分析 |
| 2 | 调节变量 | H2: 在 Z 条件下，H1 效应更强 | 分组对比 |
```

根据调研类型选择假设方向：
- **态度/学术调研**（论文主流）：假设指向态度倾向、认知程度、价值判断（量表验证）
- **行为调研**：假设指向可观测行为（频率、选择、经历）
- **满意度调研**：假设指向满意度水平及驱动因素

用 `AskUserQuestion` 确认（同样需 Unicode 转义），选项：「符合，生成问卷」或「需要调整」。

---

## 阶段三：生成问卷 + 偏差审查

用户确认假设后，同时完成以下三件事，一次性输出：

### 问卷设计规则

读取 `references/question-types.md` 了解题型选择。

**结构顺序：**
1. 筛选/预热题（确认受访者身份，建立节奏）
2. 核心测量题（量表 / 行为事实 / 满意度评分——根据假设选择）
3. 深挖题（原因、影响因素、开放追问）
4. 开放题（放最后）
5. 背景信息（人口统计，放倒数第二）

**题量控制：** 5 分钟内完成，≤15 题

**题型选择原则：**
- 验证相关/影响关系 → 李克特量表（5级或7级），成对出现（X 维度 + Y 维度）
- 行为事实 → 单选/多选，问频率、经历、选择
- 混合 → 先行为事实建立基线，再量表测态度

### 问卷星兼容格式

读取 `references/wenjuanxing-format.md` 了解完整规则。核心要点：
- 题目之间空一行，题干不得换行
- 选项不得以数字开头，一律用字母：`A.` `B.` `C.`
- 选项编号后不加空格：`A.选项文字`
- 选项文字不能有空格：`10次以上`
- 跳转说明写在题干括号内：`（Q1选D请跳过此题）`

### 偏差审查

读取 `references/bias-checklist.md` 逐题审查，必查：
- 诱导题（含价值判断词）
- 双重问题（一题问两件事）
- 假设前提题（预设了受访者行为）
- 社会期望偏差（涉及能力、收入等敏感话题）
- 选项非穷尽（缺少「不适用」或「其他」）
- 跳转逻辑缺失

### 输出三部分

**第一部分：设计意图版**（含每题的假设对应和设计说明）

**第二部分：偏差审查报告**
```markdown
## 偏差审查报告

### 发现的问题
| # | 题号 | 偏差类型 | 问题描述 | 修正建议 |

### 通过审查的题目
Q1, Q2... ✓

### 整体评估
- 偏差密度：低/中/高
- 优先修正：...
```

**第三部分：问卷星纯净版**（可直接粘贴）
```
[问卷标题]

[简短说明，1-2句]

Q1.[题干]
A.[选项文字]
B.[选项文字]
...
```

## 写入 3_调研/ 目录

生成完成后，写入文件。

**目录选择规则：**

1. 遍历 `3_调研/` 下的所有子目录（`调研1`、`调研1-xxx`、`调研2`、`调研2-xxx`……）
2. 判断这是**新建**还是**更新**：
   - **新建**：找到最大编号 N，查看是否存在空的 `/调研N`（未重命名、无文件）
     - 有空文件夹 → 将其更名为 `调研N-{本次调研的简短描述2-4字}`
     - 无空文件夹 → 新建 `调研{N+1}-{简短描述}`
   - **更新**：用户说「这是修改 XXX 调研的问卷」→ 找到对应的 `调研N-xxx` 文件夹
3. 在目标文件夹中写入：
   - 新建 → `调查问卷方案_v1.md`
   - 更新 → 检查已有版本，写入 `调查问卷方案_v{N+1}.md`，文件开头注明「基于 v{N} 修改，变更：{简述}」

**文件内容结构：**
```markdown
# {调研主题} 调查问卷方案 v{N}

生成时间：{日期}
研究假设：{一句话摘要}

---

## 研究假设

{假设表}

---

## 设计意图版问卷

{完整问卷含设计意图}

---

## 偏差审查报告

{审查报告}

---

## 问卷星纯净版

{可直接粘贴的纯净版}
```

写入后告知用户：

**任务进度（5/5）** 文件写入完成

```
已保存至 3_调研/{调研N-xxx}/调查问卷方案_v{N}.md

## 立即可做
1. 将「问卷星纯净版」内容粘贴到问卷星（wenjuanxing.com）或腾讯问卷
2. 发放收集数据（建议 150 份以上）
3. 将回收的数据文件（Excel/CSV）保存到 3_调研/{调研N-xxx}/ 目录
```

### 下一步建议

读取 `CLAUDE.md` 中的「目前进展」，从以下规则中选 1-3 条适用项建议：

- 文献导入未完成 → `/ths-paper-load` 导入文献
- 文献数量不足（< 20 篇）→ `/ths-paper-search` 补充文献
- 有数据但未完成分析 → `/ths-analyze` 分析数据
- 无提纲/草稿 → `/ths-write-now` 字数规划
- 有提纲但正文未完成 → `/ths-write-now` 继续撰写
- 全部章节已完成 → `/ths-output` 输出最终 Word 文件

---

## 快速模式

用户说「快速」「直接生成」时：

1. 自动完成第零步（从草稿/CLAUDE.md 提取，不等确认）
2. 自动推断调研目标、人群、渠道（不足部分用 **[假设]** 标注）
3. 自动生成研究假设表（不等确认）
4. 直接输出完整问卷（设计意图版 + 偏差审查 + 问卷星纯净版）
5. 写入 `3_调研/` 目录

在文件末尾汇总所有 **[假设]** 项，供用户事后核对。

---

## Gotchas

- **预期控制**：每完成一步，告诉用户当前进度，如 "**任务进度（2/5）**"
- **AskUserQuestion 中文转义**：所有中文字符串必须用 Unicode 转义（`\uXXXX`），否则报 "provided as string" 错误
- **3_调研/ 不存在时**：先创建目录再写入，`Path("3_调研/调研1").mkdir(parents=True, exist_ok=True)`
- **草稿文件多个时**：取修改时间最新的文件
- **X/Y 不明显时**：宁可多问一句，不要猜错再生成一整份问卷
