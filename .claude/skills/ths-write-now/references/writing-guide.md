## 1. 文献综述写法

### 核心定位：综合，不是罗列

文献综述的目的是展示「前人做了什么、还有什么没做、本研究为什么有价值」。不要写「A说……B说……C说……」，而要写「在X问题上，学界存在两种主要观点：一派（A、B、C）认为……，另一派（D、E）认为……，核心分歧在于……」。

**组织原则**：按主题/争议/理论框架组织，从宏观到微观，从一般到具体，逐步聚焦到本研究核心问题。

### 七类研究空白

综述必须指向本研究的必要性：

| 类型 | 说明 |
|------|------|
| 知识空白 | 某个问题尚未被研究 |
| 方法论空白 | 现有研究方法存在缺陷 |
| 实证空白 | 缺乏特定情境/行业的实证数据 |
| 理论空白 | 现有理论无法解释某现象 |
| 应用空白 | 理论未被应用于实践 |
| 人群空白 | 特定群体未被研究 |
| 时间空白 | 现有研究已过时 |

### 批判性分析要点

对每篇文献，除了描述「别人说了什么」，还要评估：
- 研究的样本/情境局限
- 结论的适用边界
- 与其他研究的矛盾或一致性

**自检问题**：读者读完综述后，能否清楚知道「前人已做了什么、还有什么没做、本研究为什么有价值」？

## 2. 因果论述：基于《为什么》

MBA论文中最常见的错误之一是将「相关」当作「因果」来写。珀尔（Judea Pearl）的因果阶梯为我们提供了清晰的判断标准。

### 因果阶梯：先判断你的证据在哪一级

| 级别 | 认知能力 | 证据类型 | 对应写法 |
|------|---------|---------|---------|
| **第一级：关联**（Seeing）| 观察到共变关系 | 问卷、访谈、财报 | 「X 与 Y 呈正相关」|
| **第二级：干预**（Doing）| 改变X后观察Y | A/B测试、准实验、DID | 「在控制其他变量后，X 使 Y 提升…」|
| **第三级：反事实**（Imagining）| 推断「如果没有X」| 需要干预级证据支持 | 「若未实施该策略，预计…」|

**绝大多数MBA论文的证据停留在第一级（调查数据），但写作语言却习惯性地跳到第二、三级。**

### 常见错误与更诚实的替代表述

| 常见错误写法 | 诚实替代写法 |
|------------|------------|
| X 导致了 Y | X 与 Y 存在正相关 |
| X 促进了 Y 的提升 | X 的增加伴随着 Y 的提升 |
| 证明了 X 对 Y 的影响 | 在一定程度上表明 X 可能与 Y 相关 |
| X 是 Y 的重要原因 | X 是与 Y 显著相关的因素之一 |
| 实施 X 可以提高 Y | 若条件与本样本类似，X 与 Y 的正向关联或可参考 |

### 混淆变量：必须承认

当 A 和 B 相关，可能是第三个变量 C 同时影响了两者。**写法模板**：
> 「本研究数据显示，X 与 Y 存在正相关（r = 0.xx，p < 0.05）。需要说明的是，本研究无法排除 [具体因素] 对两者的共同影响，因此该关联不宜直接解读为因果关系。」

### 辛普森悖论：总体数据可能误导人

总体样本的结论，在分子群组后可能完全逆转。凡是出现「分组后结论与总体不一致」时，必须在论文中专门说明，并解释哪个层次的分析更有意义。

### 反事实自检法

在写下任何因果结论前，问自己：**「如果 X 没有发生，Y 还会发生吗？」**
- 如果答案是「会」，则 X 不是 Y 的原因，需重新审视论述
- 如果无法回答，说明数据还不足以支持因果主张

**一句话总结**：诚实使用「相关」而非「导致」，不是在削弱论文，而是在保护论文不被答辩委员质疑。

## 3. 逻辑论证框架

### 论证三件套：主张-理由-证据

每个论点必须包含：
```
主张（Claim）：你要证明什么
    ↓
理由（Reason）：为什么这个主张成立
    ↓
证据（Evidence）：支撑理由的数据/事实/文献
```

**错误示例**：「该公司管理存在问题，因此绩效下降。」（主张有了，理由和证据呢？）

**正确示例**：「该公司2022-2024年净利润率从12%下降至6%（证据），与同期管理层频繁更迭（3年内4位CEO）导致的战略不一致性密切相关（理由）。根据Hambrick & Mason（1984）的高层梯队理论，管理层稳定性是战略执行连贯性的关键前提（理论支撑）。」

### 图尔敏完整论证模型（高要求时使用）

| 要素 | 说明 |
|------|------|
| 主张（Claim） | 核心论点 |
| 依据（Grounds） | 支撑主张的事实/数据 |
| 保证（Warrant） | 连接依据与主张的逻辑原则 |
| 支撑（Backing） | 支撑「保证」的理论/研究 |
| 限定（Qualifier） | 主张的适用范围 |
| 反驳（Rebuttal） | 可能的反例及应对 |

### 六类常见逻辑谬误（必须避免）

| 谬误 | 说明 | 示例 |
|------|------|------|
| 以偏概全 | 少数案例推断普遍规律 | 「访谈了3位经理，所以该行业都是这样」 |
| 循环论证 | 用结论证明结论 | 「该战略有效，因为它带来了好结果」 |
| 相关混淆因果 | 同时发生≠一个导致另一个 | 「广告投入增加与销售额同步，所以广告导致了增长」 |
| 过度推断 | 结论超出数据支撑范围 | 单一案例研究得出行业普遍结论 |
| 诉诸权威 | 只引用权威而不提供实质证据 | 「某教授说X，所以X是对的」 |
| 稻草人谬误 | 歪曲对立观点再反驳 | 故意简化竞争理论再批判 |

## 4. 数据分析章节写法

### 描述-分析-解释三层次

数据呈现不能停留在描述，必须完成三步：

```
描述（Description）→ 分析（Analysis）→ 解释（Interpretation）
```

- **描述**：这个数据/现象是什么？（「调研样本中，63%的受访者表示……」）
- **分析**：数据中有什么规律、对比、关联？（「与……相比，高出XX个百分点」）
- **解释**：这意味着什么？为什么会这样？（「这表明……的原因可能在于……」）

**常见错误**：大量呈现图表，但只写「由图可知……」，缺乏解释层。

### 方法选择必须说明 why

选择定量/定性/混合方法时，必须在论文中说明为什么选这种方法，而非其他方法。

### 三角验证

使用多种数据来源相互印证：问卷数据 + 访谈 + 二手数据。三角验证不是为了「确认」，而是为了更全面理解现象。

## 5. 商业实践章节写法

### 理论框架必须显性应用

在分析每个问题时，明确说明「根据XX理论/框架，这个现象可以解释为……」，不要让理论只是隐性地存在于背景中。

### 案例分析四项效度标准

1. **构建效度**：研究的概念是否被正确测量？
2. **内部效度**：因果关系的推断是否合理？
3. **外部效度**：结论能否推广到其他情境？
4. **信度**：分析过程是否可重复？

### 对策建议的可操作性标准

**错误**：「应该加强管理，提升效率。」

**正确**：「建议在6个月内建立跨部门项目管理委员会，由COO直接负责，每月召开战略对齐会议，以解决当前部门协作断裂问题。」

每条建议必须：
- 说明实施时间（短期0-6月 / 中期6-18月 / 长期18月以上）
- 有数据/分析支撑，不凭空提出
- 说明预期效果和衡量方式

## 6. 理论贡献与实践贡献

论文结论部分需明确区分：

- **理论贡献**：本研究对学术理论有什么新发现或新视角？（对学界的价值）
- **实践贡献**：本研究对企业管理实践有什么指导意义？（对业界的价值）

MBA论文通常更强调实践贡献。两者都需要写，但不要夸大——在结论部分诚实说明局限性（样本规模、情境特殊性、时间截面）反而是加分项。
