# ths-init

## 输入

| 输入 | 是否必须 |
|------|---------|
| 开题报告文件（PDF/Word/Markdown）/ 进行中的论文文件（Word/Markdown） / 确定的论文选题 | 必须 |

---

## 第零步：告诉用户任务基本预期

```
=================
任务：初始化论文项目（ths-init）

目标：让 Agent 理解当前论文的选题、要求、进度，并写入 claude.md。后续所有的任务都要在此基础上进行。

大约需要时间：N 分钟；
=================
```

N 的计算方式：
- 输入的是开题报告，则为 10；
- 如果是进行中的论文文件，则为15；

---

## 第一步：创建目录结构

```python
from pathlib import Path

dirs = [
    "1_开题",
    "2_参考文献",
    "3_调研/调研1",
    "4_草稿",
    "5_输出",
    "6_答辩呈现",
]
for d in dirs:
    Path(d).mkdir(parents=True, exist_ok=True)
```

如果用户提供了开题报告/进行中的论文文件且不在 `1_开题/` 目录下，帮助用户把文件移动到该目录。
遍历当前目录下原有的其他的文件/目录，读取并判断，将它们移入`2_参考文献`/`3_调研`/`4_草稿`等目录下；

---

## 第二步：生成论文格式文件

将：
`${CLAUDE_SKILL_DIR}/material/论文模板.docx` 
`${CLAUDE_SKILL_DIR}/material/论文格式.md`
复制到 `1_开题/`下）。

在汇报中注明：

> ⚠️ 当前使用内置默认 Word 模板（上海交通大学通用格式）；

---

## 第三步：解析选题信息

从开题报告/论文文件/用户输入中提取：

| 字段 | 说明 |
|------|------|
| 论文正式题目 | 完整标题 |
| 自变量 X | 题目中的影响因素/研究因素 |
| 因变量 Y | 题目中的被影响对象/研究结果 |
| 研究方法 | 从开题报告/现有文稿提取，或基于题目生成初步建议。如：问卷调查法、案例研究法、定量分析等 |
| 论文提纲 | 从开题报告/现有文稿提取，或基于题目生成初稿。注意以上只是`论文格式要求.md`中”结构要求“的正文部分，仍需要补充封面、摘要等其他内容； |
| 字数要求 | 从格式要求提取，缺省 3-4.5 万字 |

## 第四步：导入现有信息

从开题报告/论文文件/用户输入中提取“文献列表”，写入`2_参考文献/格式化参考文献列表.md`

若非开题报告，且已有部分论文，则调用 /ths-write-now 完成字数分配并将现有内容写到 `4_草稿`作为第一版正文；

同时，依据以下信息更新 {目前进展} ：

[ ] 完成文献导入：`2_参考文献/格式化参考文献列表.md` 中的内容都存在于`2_参考文献/参考文献列表.xml`则 [✅]
[ ] 有足够文献：`2_参考文献/参考文献列表.xml` 中有 >= 20 篇文献则 [✅]
[ ] 有调研方案 `3_调研/参考文献列表.xml` 中有至少一篇方案，或`4_草稿`中已有调研数据则 [✅]
[ ] 完成调研分析 `3_调研/参考文献列表.xml` 中有至少一篇分析结论，或`4_草稿`中已有调研数据则 [✅]
[ ] 完成章节规划 `4_草稿`中已有草稿或正文 [✅]
[ ] 完成全部章节 `4_草稿`中最新一篇正文已完成全部章节 [✅]

---

## 第五步：提取文本风格

如果用户给出了开题报告或论文正文，从中提取以下信息到{文本风格}：
- 词汇偏好
- 句式偏好
- 标点偏好
- 分段偏好
- 典型参考段落；
（如果用户只给了标题，那这段暂无）

## 第六步：写入 CLAUDE.md

在项目根目录的 `CLAUDE.md` 中追加以下内容（如文件不存在则创建）：

```markdown
# 论文项目信息

## 选题

- **论文题目**：{论文正式题目}
- **自变量（X）**：{X}
- **因变量（Y）**：{Y}

## 研究方法

{研究方法}

## 论文提纲

{提纲，章节结构}

## 文本风格

{文本风格}

## 字数要求

{字数要求}

## 目前进展

{目前进展}

## 项目目录结构

- **1_开题/**：开题报告文件
- **2_参考文献/**：
  - 包含论文引用资料的下载版 PDF
  - `参考文献列表.xml`：引用资料索引
  - `格式化参考文献列表.md`：GBT-7714 格式引用列表
- **3_调研/**：
  - 子文件夹为 调研1、调研2……
  - 每个子文件夹包含：调研方案（问卷/访谈提纲/实验方案）、结果数据、结果分析文件、结果分析网页
- **4_草稿/**：
  - 论文提纲文件，以 `提纲_` 开头
  - 论文正文，以 `正文_` 开头
- **5_输出/**：论文输出 Word 文件
- **6_答辩呈现/**：答辩 PPT 及相关材料
```

追加完选题信息后，在 CLAUDE.md 末尾继续追加角色配置：

```python
from pathlib import Path

role_src = Path("${CLAUDE_SKILL_DIR}/material/角色.md")
claude_md = Path("CLAUDE.md")

with open(claude_md, "a", encoding="utf-8") as f:
    f.write("\n\n---\n\n")
    f.write(role_src.read_text(encoding="utf-8"))
```

这样后续所有对话都能感知「老魏教授」的角色设定。

## 完成，向用户汇报

```
## 论文项目初始化完成
----

**论文题目**：{题目}
**X → Y**：{X} → {Y}
**研究方法**：{方法}

### 已完成
- [x] CLAUDE.md 写入选题、提纲、目录结构说明
- [x] 创建目录：1_开题/ 2_参考文献/ 3_调研/ 4_草稿/ 5_输出/ 6_答辩呈现/
- [x] 论文格式文件：1_开题/论文模板.docx 与 论文格式.md（来源：{用户提供 Word 模板 / 默认交通大学模板}）

### 下一步建议
1....
2....
```

下一步建议规则（给出1-3项目）：
参考 `claude.md` 中的目前进展章节。
- 未完成文献导入：建议 /ths-paper-load 导入
- 没有足够文献：建议 /ths-paper-search 搜索文献
- 没有调研方案
  - 但`claude.md`的研究方法中有调查问卷：建议 /ths-questionaire 进行问卷设计；
  - 但`claude.md`的研究方法中没有调查问卷：建议直接讨论调研方法的设计；
- 未完成调研分析：建议 /ths-analyze 进行分析；
- 未完成章节规划：建议 /ths-write-now 进行章节规划；
- 未完成全部章节：建议 /ths-write-now 进行下一章撰写；
- 已完成全部章节：建议 /ths-review 进行 AI 审阅，或 ths-output 进行最终输出。

---

## Gotchas

- **预期控制**：每完成一步，告诉用户当前进度，如 "**任务进度(5/10)**"
- **CLAUDE.md 追加不覆盖**：如已有内容，在文件末尾追加，不替换已有内容
- **目录已存在时不报错**：`exist_ok=True`
- **开题报告路径**：如果用户提供的是绝对路径，原样使用；如果是相对路径，相对于项目根目录
- **X/Y 提取**：如果题目结构不明显（如"某公司管理优化研究"），请用户确认 X 和 Y，不要猜测
