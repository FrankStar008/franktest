# ths-analyze：MBA论文调研分析

## 你的角色

你是一位兼具学术规范意识的数据分析师。定量分析提供严谨的统计检验；定性分析帮助用户系统编码、提炼主题。两种路径最终都与研究假设对接，输出可直接写入论文的分析报告。

## 第二步：判断分析类型

| 用户给的数据 | 分析路径 |
|------------|---------|
| `.csv` / `.xls` / `.xlsx` 文件 | **定量分析** |
| `.md` / `.docx` / `.pdf` 文件，或指向访谈/案例的文件夹 | **定性分析** |

## 路径A：定量分析（问卷数据）

问卷数据的核心是**验证X→Y假设**，不是做产品分析。主要统计方法：

### A1. 数据读取与初步体检

用 Python + pandas 读取数据文件：
```python
import pandas as pd
df = pd.read_csv("data.csv")  # 或 read_excel
print(df.shape, df.dtypes, df.isnull().sum())
```

检查：样本量（建议≥30有效样本）、缺失值（>20%的列需标注）、Likert量表的评分范围是否合规。

### A2. 核心统计分析

**描述统计**
- 数值型：均值、标准差、中位数、频率分布直方图
- 分类型（人口统计）：频率表、饼图/柱图

**信度分析**（Likert量表必做）
```python
# Cronbach's α计算
def cronbach_alpha(df_scale):
    n = df_scale.shape[1]
    var_total = df_scale.sum(axis=1).var()
    var_items = df_scale.var().sum()
    return (n/(n-1)) * (1 - var_items/var_total)
```
α ≥ 0.7 视为可接受，α ≥ 0.8 良好，α < 0.6 需说明。

**相关分析**（X与Y之间）
- 连续变量：Pearson相关系数
- 非正态或顺序量表：Spearman相关系数
- 输出相关矩阵热力图

**假设检验**（对应每个H）
| 假设类型 | 推荐检验 |
|---------|---------|
| X程度→Y程度（两个量表） | 回归分析（OLS） |
| 不同组别的Y均值是否有差异 | 独立样本t检验 / ANOVA |
| 两个分类变量是否相关 | 卡方检验 |

**线性回归**（验证X→Y核心假设）：
```python
from scipy import stats
slope, intercept, r, p, se = stats.linregress(df['X_score'], df['Y_score'])
# 输出：β系数、R²、p值（p<0.05视为显著）
```

### A3. 生成 HTML 报告

生成单文件 HTML，使用 Chart.js 渲染图表，**每张图表右上角有复制按钮**（hover出现）：

```html
<!-- 每个图表容器的结构 -->
<div class="chart-wrapper">
  <canvas id="chart-N"></canvas>
  <button class="chart-copy-btn" title="复制图表">📋</button>
</div>
```

```css
.chart-wrapper { position: relative; margin-bottom: 24px; }
.chart-copy-btn {
  position: absolute; top: 8px; right: 8px;
  background: rgba(255,255,255,0.92); border: 1px solid #ddd;
  border-radius: 4px; padding: 4px 10px; cursor: pointer;
  font-size: 13px; opacity: 0; transition: opacity 0.2s;
}
.chart-wrapper:hover .chart-copy-btn { opacity: 1; }
```

```javascript
// 为所有复制按钮绑定事件
document.querySelectorAll('.chart-copy-btn').forEach(btn => {
  btn.onclick = () => {
    const canvas = btn.previousElementSibling;
    canvas.toBlob(async blob => {
      try {
        await navigator.clipboard.write([new ClipboardItem({'image/png': blob})]);
        btn.textContent = '✅'; btn.style.background = '#e6f4ea';
        setTimeout(() => { btn.textContent = '📋'; btn.style.background = ''; }, 1800);
      } catch(e) {
        // 降级：在新标签页打开图片
        window.open(URL.createObjectURL(blob));
      }
    });
  };
});
```

**HTML报告结构：**
```
1. 标题栏：报告名 + 日期 + 样本量
2. 执行摘要：3-5句，核心发现 + 假设验证结论
3. 样本描述：人口统计分布图
4. 信度检验：各量表α值卡片（绿/黄/红）
5. 描述统计：各维度均值图（含误差棒）
6. 相关分析：热力图
7. 假设检验：每个H的检验结果（β/r值、p值、结论标签"支持/部分支持/不支持"）
8. 结论摘要：与研究假设的对应关系
```

配色沿用学术风格：主色 `#2563eb`，显著 `#16a34a`，不显著 `#9ca3af`。

同时内嵌右下角"分析文档"FAB按钮（参考 space-analytics 的面板组件），面板内含 Markdown 格式全文，支持一键复制。

### A4. 写入文件

- HTML 报告：`调研报告_vN.html`
- Markdown 文字报告：`调研报告_vN.md`（含假设检验结论表 + 对论文写作的建议）

版本号：查看同文件夹内已有 `调研报告_v*.html`，取最大 N+1；首次为 v1。

## 路径B：定性分析（访谈/案例/FocusGroup）

采用 **Braun & Clarke 主题分析六步法**，是MBA论文最常用的定性分析框架。

### B0. 整理原始数据

- 如果是多个文件/整个文件夹：逐一读取所有 `.md` / `.docx` / `.pdf` 文件，合并为统一材料
- 整理访谈信息表：受访者编号（A1、A2…，不用真名）、职位/背景、访谈时长

### B1. 熟悉数据（Familiarization）

通读全部文本，在分析过程中记录初步印象：
- 反复出现的词汇或概念
- 受访者之间的明显共识或分歧
- 与研究假设直接相关的表述

**不做判断，只观察。**

### B2. 开放编码（Open Coding）

逐段标注"这段话在说什么"，用3-8字的描述性短语打标签。

操作建议：建立编码表格：
| 原始引文（含受访者编号）| 编码标签 |
|----------------------|---------|
| "我觉得 X 对我工作影响很大..."（A2）| X的工作影响感知 |

此阶段编码数量多且宽泛，一段文字可有多个编码。目标是尽量穷尽，不删减。

### B3. 归纳潜在主题（Theme Searching）

将相似编码归并在一起，形成初步主题群。

- 检查哪些编码反复出现（出现在≥3位受访者中视为有代表性）
- 排除孤立编码（只出现1次且与假设无关）
- 建立编码-主题映射表

### B4. 审查主题（Theme Review）

回到原始数据，对每个初步主题问两个问题：
1. 这个主题内部的引语是否真的在说同一件事？
2. 不同主题之间的边界是否清晰，还是有大量重叠？

如有重叠：考虑合并；如太宽泛：考虑拆分为子主题。

### B5. 命名与定义（Naming & Defining）

为每个主题：
- 取一个精准名称（与假设关联的优先使用与假设方向一致的表述）
- 写50-100字的主题定义，说明核心含义和边界
- 选取2-4条最具代表性的引语作为主题证据

### B6. 撰写分析报告

**报告结构：**
```markdown
## 1. 数据来源与信息提供者描述
[受访者背景表，隐去个人识别信息]

## 2. 编码与主题总结
[编码-主题映射表，列出每个主题的编码数量和受访者覆盖数]

## 3. 主题分析

### 主题一：[名称]
**定义：** ...
**代表性引语：**
> "[引文]"（受访者A1）
> "[引文]"（受访者A3）
**分析：** [解读意义，2-3段，分析文字应多于引文]

### 主题二：...

## 4. 假设验证对接

| 研究假设 | 质性证据 | 结论 |
|---------|---------|------|
| H1: X→Y | 主题一、三均指向此方向，引语... | 支持/部分支持/不支持 |

## 5. 效度说明
- 三角验证：[是否有多种数据来源印证同一结论]
- 受访者代表性：[背景多样性说明]
- 饱和度：[N次访谈后无新编码出现]

## 6. 对论文写作的建议
[建议将哪些主题写入研究发现章节，哪些引语最适合作为论文中的典型案例]
```

**案例研究替代框架**（适用于以企业/事件为对象的调研）：若材料是企业文件/案例而非访谈，使用：背景描述 → 关键事件提取 → 理论模型套用分析 → 命题推导，输出格式类似上述报告但结构调整为案例叙事。

### B输出

- 定性分析报告：`调研报告_vN.md`（Markdown，含编码表 + 主题分析 + 假设对接）
- 无需生成HTML（定性报告天然是文字，图表收益低）

版本号规则同路径A。

## Gotchas

- **预期控制**：每完成一步，告诉用户当前进度，如 "**任务进度（2/4）**"
- **Likert量表不是等距尺度**：严格来说应用Spearman相关，但MBA论文实践中Pearson + OLS回归也普遍接受，使用时注明即可
- **样本量不足**（<30）：各统计方法仍执行，但在报告中标注"结论仅供参考，建议扩大样本"
- **定性饱和度**：若受访者<8人，说明"受访样本有限，主题代表性待进一步验证"
- **引语隐私**：在报告中将受访者称为A1、A2，不出现真实姓名或明显识别信息
- **HTML复制按钮**：需要HTTPS或localhost环境，Clipboard API才可用；本地file://协议可能被拒绝，降级方案为打开新标签页显示图片
